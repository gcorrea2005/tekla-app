1. Descripcion General

Tekla Connection Manager es una aplicacion web que permite aplicar conexiones estructurales (placas de cortante, placas finales, angulos de clip, etc.) a elementos de un modelo de Tekla Structures directamente desde el navegador.

Funcionalidades Principales

  • Verificacion de conexion con Tekla Structures en tiempo real
  • Listado de todos los elementos del modelo (vigas y columnas)
  • Aplicacion de conexiones individuales con parametros personalizados
  • Aplicacion en lote — genera conexiones automaticamente entre vigas y columnas
  • Catalogo de 7 tipos de conexion predefinidos
  • Resultados detallados con estado exitoso/fallido por conexion

3. Inicio Rapido

1

Abrir Tekla Structures 2020

Abre Tekla Structures y carga el modelo con el que deseas trabajar. El modelo debe contener al menos vigas o columnas.

2

Iniciar el Backend (Flask)

Abre una terminal en la carpeta server/ y ejecuta:

cd C:\Users\gcorrea\Desktop\prueba\tekla-app\server
python app.py

El servidor arrancara en http://localhost:3001.

3

Iniciar el Frontend (Vite)

Abre otra terminal en la carpeta client/ y ejecuta:

cd C:\Users\gcorrea\Desktop\prueba\tekla-app\client
npm run dev

La interfaz estara disponible en http://localhost:5176.

4

Abrir en el Navegador

Navega a http://localhost:5176. Veras el estado de conexion en la barra superior.

4. Interfaz de Usuario

4.1 Barra de Estado (StatusBar)

Parte superior de la pantalla. Muestra:

  • Titulo: "Tekla Connection Manager"
  • Estado: Conectado o Desconectado
  • Nombre del modelo: Se muestra cuando hay conexion activa
  • Boton "Actualizar": Refresca el estado de conexion y recarga los elementos

4.2 Tabla de Elementos (BeamTable)

Muestra todos los elementos estructurales del modelo de Tekla.

ColumnaDescripcion
IDIdentificador unico del elemento en Tekla
NombreNombre asignado al elemento
TipoViga o Columna
PerfilSeccion transversal (ej: IPE300, HEB200)
MaterialMaterial del elemento (ej: S275, S355)

Puedes seleccionar elementos haciendo clic en la fila o en el checkbox. Los elementos seleccionados se resaltan.

4.3 Formulario de Conexion (ConnectionForm)

Para aplicar una conexion individual. Contiene:

  • Elemento Primario: Generalmente la columna (soporte)
  • Elemento Secundario: Generalmente la viga (miembro conectado)
  • Tipo de Conexion: Catalogo de 7 componentes disponibles
  • Tamano de Tornillo: M12, M16, M20 o M24
  • Espesor de Placa: Entre 6mm y 40mm

4.4 Panel de Lote (BatchPanel)

Para aplicar conexiones de forma masiva. Genera automaticamente conexiones Shear Tab entre todas las vigas y columnas del modelo. Muestra un resumen con total, exitosas y fallidas.

5. Flujo de Trabajo

5.1 Conexion Individual

  1. Haz clic en "Cargar Elementos" para obtener la lista de vigas y columnas
  2. Cambia a la pestana "Conexion Individual"
  3. Selecciona el Elemento Primario (columna/soporte)
  4. Selecciona el Elemento Secundario (viga/miembro)
  5. Elige el Tipo de Conexion del catalogo
  6. Ajusta los parametros: tamano de tornillo y espesor de placa
  7. Haz clic en "Aplicar Conexion"
  8. El resultado aparecera en la barra de estado

5.2 Aplicacion en Lote

  1. Haz clic en "Cargar Elementos"
  2. Cambia a la pestana "Aplicacion en Lote"
  3. Revisa el conteo de columnas, vigas y conexiones estimadas
  4. Haz clic en "Generar y Aplicar Todas las Conexiones"
  5. Espera a que se procesen todas las conexiones
  6. Revisa el resumen de resultados (exitosas vs fallidas)
Advertencia: La aplicacion en lote genera una conexion Shear Tab (#44) por cada combinacion viga-columna. Si tienes 10 vigas y 5 columnas, se crearan 50 conexiones. Verifica que esto es lo que deseas antes de ejecutar.

6. Tipos de Conexion

#NombreCategoriaUso Tipico

7. Aplicacion en Lote — Detalle

El panel de lote crea automaticamente conexiones Shear Tab (#44) entre cada par columna-viga encontrado en el modelo.

Logica de Generacion

para cada columna en el modelo:
    para cada viga en el modelo:
        aplicar conexion(columna.id, viga.id, #44, "Shear Tab")

Esto genera N_columnas × N_vigas conexiones.

Interpretacion de Resultados

EstadoSignificado
OKConexion aplicada exitosamente en Tekla
ERRORFallo al aplicar — ver columna "Mensaje" para detalles

8. Referencia API

El backend expone los siguientes endpoints REST en http://localhost:3001:

MetodoEndpointDescripcion
GET /api/status Estado de conexion con Tekla
GET /api/beams Listar todas las vigas
GET /api/columns Listar todas las columnas
GET /api/objects Listar todos los elementos (vigas + columnas)
GET /api/components Catalogo de tipos de conexion
POST /api/connections Aplicar una conexion individual
POST /api/connections/advanced Aplicar conexion con parametros avanzados
POST /api/connections/batch Aplicar multiples conexiones en lote
DELETE /api/connections/:id Eliminar una conexion por ID

Ejemplo: Aplicar Conexion

POST /api/connections
Content-Type: application/json

{
  "primaryId": 12345,
  "secondaryId": 67890,
  "componentNumber": 44,
  "componentName": "Shear Tab",
  "boltSize": "M16",
  "plateThickness": 10
}

Respuesta Exitosa

{
  "success": true,
  "message": "Conexion aplicada correctamente"
}

9. Solucion de Problemas

ProblemaCausa ProbableSolucion
"Desconectado" en la barra de estado Tekla Structures no esta abierto o no hay modelo cargado Abre Tekla 2020 y carga un modelo. Haz clic en "Reintentar Conexion"
"No hay elementos cargados" No se ha hecho clic en "Cargar Elementos" Haz clic en el boton "Cargar Elementos" de la tabla
Error al aplicar conexion IDs invalidos o elementos incompatibles Verifica que los IDs existen y que los elementos son geometricamente compatibles
Puerto 3001 en uso Otra instancia del servidor esta corriendo Cierra la instancia anterior o cambia el puerto en .env
Puerto 5176 en uso Otra instancia de Vite esta corriendo Vite buscara automaticamente el siguiente puerto disponible
pythonnet no encuentra los DLLs Ruta incorrecta a los plugins de Tekla Verifica que Tekla 2020 esta en C:\Program Files\Tekla Structures\2020.0\
