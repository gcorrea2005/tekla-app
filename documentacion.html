1. Arquitectura

La aplicacion usa edge-js para cargar los DLLs de Tekla Structures directamente desde Node.js, eliminando la necesidad de un proyecto .NET separado.

React + Vite (puerto 5173)
↓ ↑ HTTP (axios)
Express API (puerto 3001)
↓ ↑ edge-js (C# inline)
Tekla Open API DLLs
↓ ↑
Tekla Structures 2020

Clave: edge-js permite ejecutar codigo C# inline dentro de Node.js. Cada modulo en server/tekla/ contiene funciones C# que llaman a la API de Tekla. edge-js serializa automaticamente los objetos entre JavaScript y .NET.

3. Requisitos

  • Tekla Structures 2020 instalado (con los DLLs de la API)
  • Node.js 18+ (LTS recomendado)
  • npm o yarn
  • .NET Framework 4.7.2+ (necesario para edge-js, pero NO necesitas crear un proyecto .NET)

Importante: edge-js necesita .NET Framework instalado en Windows para poder cargar los DLLs de Tekla. Esto NO significa que necesites crear un proyecto .NET, solo que el runtime debe estar presente (ya viene con Windows).

4. Instalacion

4.1 Instalar dependencias

cd Desktop/prueba/tekla-app
npm run install:all

Este comando instala las dependencias del root, server y client.

4.2 Configurar ruta de Tekla

Edita server/.env con la ruta correcta a tu instalacion de Tekla:

TEKLA_BIN_PATH=C:\Program Files\Tekla Structures\2020.0\nt\bin

4.3 Ejecutar

1

Abrir Tekla Structures 2020 con un modelo que contenga elementos.

2

Iniciar la app:

npm run dev

Esto arranca Express en :3001 y Vite en :5173.

3

Abrir http://localhost:5173 en el navegador.

5. Uso de la App

5.1 Verificar Conexion

Al abrir la app, muestra automaticamente si esta conectado a Tekla. Si no, haz clic en "Reintentar Conexion".

5.2 Cargar Elementos

Haz clic en "Cargar Elementos" para obtener todas las vigas y columnas del modelo activo.

5.3 Conexion Individual

  1. Selecciona el elemento primario (columna o viga)
  2. Selecciona el elemento secundario
  3. Elige el tipo de conexion (Shear Tab, End Plate, etc.)
  4. Configura parametros (tornillo, espesor de placa)
  5. Haz clic en "Aplicar Conexion"

5.4 Aplicacion en Lote

  1. Cambia a la pestana "Aplicacion en Lote"
  2. Haz clic en "Generar y Aplicar Todas las Conexiones"

    3. Crea automaticamente conexiones Shear Tab entre todas las vigas y columnas.

7. Como funciona edge-js

edge-js permite ejecutar codigo C#/.NET directamente desde Node.js. Los DLLs de Tekla se cargan como assemblies .NET.

Estructura de un modulo

const edge = require('edge-js');
const path = require('path');

// 1. Cargar assemblies
edge.assemblies([
  path.join(TEKLA_BIN, 'Tekla.Structures.Model.dll'),
]);

// 2. Definir funcion C# inline
const miFuncion = edge.func({
  source: function () {/*
    using System.Threading.Tasks;
    using Tekla.Structures.Model;

    public class Startup
    {
        public async Task<object> Invoke(object input)
        {
            // Codigo C# aqui
            var model = new Model();
            return new { connected = model.GetConnectionStatus() };
        }
    }
  */}
});

// 3. Llamar desde JS
const resultado = await miFuncion({});
console.log(resultado.connected);

Nota: El C# va dentro de un comentario /* */ que edge-js parsea. La clase debe llamarse Startup con un metodo Invoke que recibe object input y retorna Task<object>.

9. Solucion de Problemas

"Could not load assembly"
Verifica que TEKLA_BIN_PATH en server/.env apunte a la carpeta correcta de Tekla.

"No conectado a Tekla"
Asegurate de que Tekla Structures este abierto con un modelo cargado antes de iniciar el servidor.

"edge-js install error"
Ejecuta npm install con permisos de administrador. edge-js compila codigo nativo durante la instalacion.

El servidor no inicia en el puerto 3001
Cambia PORT en server/.env y actualiza el proxy en client/vite.config.js.
